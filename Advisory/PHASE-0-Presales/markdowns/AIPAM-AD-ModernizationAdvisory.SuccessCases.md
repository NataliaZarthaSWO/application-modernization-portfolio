## Slide 9
1. KFC in Ecuador. At SWO, we have designed a comprehensive strategy to facilitate the use and adoption of Artificial Intelligence (AI) throughout the KFC MaxPoint team. The plan begins by establishing a governance model led by an AI Architect. Their purpose will be to ensure the responsible use of technology within the project, based on the following functions: Training and knowledge transfer. Management and provisioning of AI services and tools. Monitoring of performance KPIs. Alignment of the SWO strategy with KFC corporate guidelines. Furthermore, we have defined specific use cases according to each role within the product lifecycle: Functional: Prompt engineering for drafting user stories. Architects: Utilization of technical definition agents, documentation, architecture guidelines, and system security reviews. Developers: Access to enabled IDEs (VS Code or Antigravity) and the use of prompts for task planning, execution, and validation. QA: Implementation of OneClick Test. DevOps: Use of prompts to accelerate the generation and validation of deployment pipelines. To enhance these processes, the strategy includes a RAG system that will allow for scaling toward an ecosystem of specialized agents in technical and management areas. Finally, this proposal has already been shared with the KFC team, including progress from the PoC that demonstrates its effective integration into the current workflow.  
  
Jetison Barajas

## Slide 10
2. Success Story 2 Type: Solution Development Challenge The client required the development of a web portal to manage Bepensa’s Middleware, built under a Cloud Native architecture with an Angular SPA frontend, strictly adhering to the defined UX/UI design and architectural standards. The project had significant time and budget constraints, with an estimated maximum timeline of 3 to 4 months. Solution Frontend development was accelerated by leveraging Figma MCP for automated UI component generation, complemented by specialized prompts to create baselines and iterative versions for each required form. In parallel, a robust quality assurance process was established, including automated unit and E2E testing, ensuring consistency, traceability, and architectural compliance. Benefits Achieved Standardized design implementation from Figma. Significant acceleration in UI component development. Full adherence to the defined frontend base architecture. Improved code quality through automated unit and E2E testing. Reduced development effort and shorter product delivery time.   
  
Javier Vargas

## Slide 11
​  
Success Story 3 Type: Solution Assessment and AI-Accelerated Refactoring Challenge The client Colegios Peruanos needed to efficiently analyze the Innova Schools Online Admission System to define modernization strategies that would ensure a shorter time-to-market and accelerate their regional digital transformation process over AWS. Solution Generative AI agents were integrated for source code analysis, combined with MCPs from static code analysis and database analysis tools. This enabled the identification of technical blockers, modernization opportunities, and accelerated solution paths (One Click model). Additionally, AI-augmented code transformation tools were incorporated for .NET applications, aimed at reducing technological obsolescence and accelerating cloud migration (using AWS Transform). Benefits Achieved Clear and well-founded definition of the modernization strategy and target architecture. Rapid discovery of the application's technical status and key modernization gaps (within hours). Acceleration of refactoring and containerization processes. Reduction in licensing and assessment-related operational costs. Decreased uncertainty regarding the technical feasibility of modernizing the application ecosystem.​  
  
Javier Vargas
